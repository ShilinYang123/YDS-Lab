## 🏗️ 架构概述

YDS-Lab采用AI原生的分布式协作架构设计，通过Trae IDE集成多智能体系统，结合标准化的目录结构和自动化工具链，实现"一个人+一群AI"的高效协作模式。

核心特色是**项目生命周期管理体系**：从任务规划（02-task/）→开发实现（03-dev/）→生产部署（04-prod/），实现全流程管控。同时通过01-struc/组织架构模拟真实公司运作，提供完整的AI协作环境。

## 🔧 核心模块说明

### 🔄 项目生命周期管理体系（V5.0新版）

#### 📁 02-task/ - 任务规划与工作区
- **用途**：项目规划和任务分配的工作空间
- **命名规范**：`00x-任务名称`（如`001-长记忆系统开发`）
- **内容**：任务书、实施方案、项目规划文档
- **流程**：需求分析 → 任务分解 → 资源分配 → 开发计划

#### 📁 03-dev/ - 项目开发工作区
- **用途**：项目开发阶段的工作空间
- **命名规范**：与02-task保持一致的`00x-项目名称`
- **内容**：源代码、开发文档、测试数据、调试信息
- **流程**：架构设计 → 编码实现 → 单元测试 → 集成测试

#### 📁 04-prod/ - 生产环境部署区
- **用途**：项目生产环境的部署配置
- **命名规范**：与02-task、03-dev保持一致的`00x-项目名称`
- **内容**：生产配置、部署脚本、监控配置、运维文档
- **流程**：环境配置 → 部署测试 → 性能优化 → 监控告警

### 🏢 01-struc/ - 公司组织架构

YDS AI公司的组织架构模拟，采用阴阳五行治理理念：

#### 阴阳五行架构

**🌓 一阴一阳之谓道 - 中华文化命名特例**

0A-board-chair（董事长）与0B-general-manager（总经理）的命名设计体现了中华文化的深刻智慧：

- **0A-board-chair（阴）**: 董事长 - 战略决策与监督
  - "A"在前，象征"阴生阳长"的自然规律
  - 阴主静，主内，主决策与监督，如月之阴面，沉稳内敛
  - 董事长负责战略方向的把控，如太极之阴，为万物之始

- **0B-general-manager（阳）**: 总经理 - 战略执行与落地  
  - "B"在后，承接阴之生，体现阳之成
  - 阳主动，主外，主执行与落地，如日之阳面，积极外显
  - 总经理负责战略的执行实施，如太极之阳，推动万物生长

这一阴一阳的配置，不仅是命名的艺术，更是治理哲学的体现：阴为体，阳为用；阴主战略，阳主战术；阴如根，阳如枝。两者相辅相成，构成完整的治理体系。

**🌳 五行相生 - 部门职能配置**
- **木（01-planning）**: 企划部 - 战略规划与品牌设计
- **火（02-finance）**: 财务部 - 财务分析与预算管理
- **土（03-resource-admin）**: 资源与行政部 - 人力资源与合规管理
- **金（04-dev-team）**: 开发部 - 技术架构与系统设计
- **水（05-marketing）**: 市场部 - 市场洞察与营销策略

#### 🏢 三级存储规范（V5.1新增）
系统采用严格的四级存储架构，确保文件管理规范化：

**📁 bak/ - 一级备份存储**
- 全公司只建立一级备份目录：`S:\YDS-Lab\bak`
- 所有备份必须集中存储，不允许零散备份
- 禁止备份膨胀，统一备份策略和管理

**📁 docs/ - 三级文档存储**
- **公司级文档**：`01-struc/docs/` - 存放公司级战略规划、组织流程等核心文档
- **项目级文档**：`项目名/docs/` - 各项目的专用文档存储
- **系统级文档**：`docs/` - 系统维护和其他通用文档

**📁 logs/ - 三级日志存储**  
- **公司级日志**：`01-struc/logs/` - 公司级操作日志、系统日志
- **项目级日志**：`项目名/logs/` - 各项目的运行日志
- **系统级日志**：`logs/` - 系统维护和通用日志

**📁 rep/ - 三级报告存储**
- **公司级报告**：`01-struc/rep/` - 公司级质量报告、审计报告等
- **项目级报告**：`项目名/rep/` - 各项目的专项报告
- **系统级报告**：`rep/` - 系统维护和通用报告

### ⚙️ 简化工具集

#### 📁 tools/ - 核心工具集（简化版）
全局运维和开发工具，采用极简设计：
- **ch.py**: 检查工具（check）- 环境检查与健康监控
- **fi.py**: 文件工具（file）- 文件处理与管理
- **st.py**: 状态工具（status）- 系统状态监控
- **up.py**: 更新工具（update）- 结构与配置更新

p#### 📁 venv-tools/ - Python虚拟环境管理
Python虚拟环境专用目录，用于隔离项目依赖：
- **pyvenv.cfg**: 虚拟环p.境配置文件
- **Lib/**: Python库文件存储
- **Scripts/**: 可执行脚本
- **Include/**: C语言头文件
- **环境隔离**: 确保不同项目依赖不冲突
- **版本控制**: 精确控制Python版本和包版本
- **系统保护**: 避免污染系统Python环境

#### 📁 monitoring/ - 系统监控与告警
系统监控基础设施，提供实时监控能力：
- **grafana/**: 可视化监控面板
- **prometheus/**: 指标收集与存储
- **logs/**: 监控日志存储
- **告警配置**: 自动告警规则设置
- **性能监控**: 系统性能指标跟踪
- **健康检查**: 服务状态实时监控

#### 📁 node_modules/ - Node.js依赖管理
Node.js项目依赖包存储目录：
- **JavaScript包**: 前端和后端依赖库
- **构建工具**: Vite、TypeScript等构建工具
- **开发依赖**: 开发环境所需工具
- **运行时依赖**: 生产环境所需库
- **版本锁定**: package-lock.json确保版本一致性
- **模块缓存**: 本地缓存加速后续安装

#### 📁 training/ - 培训与学习资源
AI协作平台培训和学习材料：
- **Exercises/**: 实践练习题库
- **Materials/**: 培训大纲和学习材料
- **理论知识**: 系统架构和概念介绍
- **实践操作**: 智能体和工具使用练习
- **故障排除**: 常见问题解决方案
- **项目作业**: 综合应用练习

#### 📁 config/ - 全局配置中心
统一配置管理，避免配置分散：
- **structure_config.yaml**: 目录结构配置
- **production.yaml**: 生产环境配置
- **rbac_config.json**: 权限控制配置

## 📋 使用指南

### 新项目创建
```bash
# 创建新任务
python tools/up.py --create-task --name "新项目名称"

# 创建开发项目
python tools/up.py --create-dev --id 004

# 创建生产部署
python tools/up.py --create-prod --id 004
```

### 环境检查
```bash
# 系统状态检查
python tools/ch.py --status

# 结构合规检查
python tools/ch.py --structure

# 配置文件检查
python tools/ch.py --config
```

### 目录结构更新
```bash
# 更新结构清单（候选模式）
python tools/up.py --structure --candidate

# 发布正式清单（需审批）
python tools/up.py --structure --release
```

### 文件处理
```bash
# 编码检查
python tools/fi.py --encoding-check

# 文件整理
python tools/fi.py --organize

# 备份管理
python tools/fi.py --backup
```
