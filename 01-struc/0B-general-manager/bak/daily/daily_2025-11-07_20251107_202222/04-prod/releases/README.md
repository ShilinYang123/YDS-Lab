# YDS-Lab Output 目录结构说明

## 📁 目录概述

`output/` 目录是 YDS-Lab 的**成果管理中心**，采用 `00x` 序号格式进行标准化管理，统一存放：
- 系统生成的输出文件和报告
- 客户项目的最终交付成果
- 项目展示材料和演示文档

## 🗂️ 目录结构规范

### 序号格式：`00x-项目名称`
- **001-009**：具体项目成果
- **008**：系统报告
- **009**：临时输出
- **010**：展示材料

## 📝 使用规范

### 1. 新项目添加
- 按序号顺序分配：`011-新项目名称`
- 项目名称使用英文或拼音，避免特殊字符
- 每个项目目录下建议包含：
  - `README.md` - 项目说明
  - `deliverables/` - 交付物
  - `documentation/` - 相关文档
  - `assets/` - 资源文件

### 2. 文件管理
- **最终成果**：放入对应项目目录
- **临时文件**：使用 `009-temp-outputs`
- **系统报告**：使用 `008-system-reports`
- **展示材料**：使用 `010-showcase-materials`

### 3. 清理策略
- 定期清理 `009-temp-outputs` 中的过期文件
- 重要成果及时备份到 `../Backups/` 目录
- 客户交付后，保留副本用于后续维护

## 🔄 与其他目录的关系

```
YDS-Lab/
├── projects/          # 项目开发过程
├── output/           # 项目最终成果 ← 当前目录
├── Production/       # 生产环境部署
└── Backups/         # 重要文件备份
```
