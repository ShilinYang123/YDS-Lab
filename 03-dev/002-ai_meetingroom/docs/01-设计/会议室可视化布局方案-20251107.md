标题：会议室可视化布局方案（根据示意图实现）

目标
- 实现“左侧会议桌与参会人分布 + 右侧资料播放展示白板 + 发言人文字展示区”的简洁直观布局。
- 增加“语音开关”，可一键控制系统朗读（TTS）开/关。

页面结构（主要区域）
- 左侧：参会人员列表（含头像、角色、在线状态）。
- 中部：会议桌可视化区（主持人、发言人座位联动；演示文稿可采用叠加层展示）。
- 右上：资料播放展示白板（用于图片、PDF预览或HTML内容展示）。
- 右下：发言人文字展示区（展示最新的发言文本，突出当前发言人姓名与颜色）。
- 右侧附加：聊天面板、白板面板、演示文稿列表面板（保持原有功能，便于上传与沟通）。

交互说明
- 发言联动：当收到 newMessage 时，聊天面板记录消息，同时“发言人文字展示区”跟随更新。
- 展示联动：点击演示文稿列表的“展示”按钮，内容同时出现在中部叠加层与右上“展示白板”。
- 语音开关：会议控制栏新增“语音开关”按钮，默认开启；关闭后不再调用浏览器 SpeechSynthesis（TTS）。
- 语音指示器：发言时在其座位头像上显示动态指示点，3秒后自动消失。

技术实现要点
- CSS新增：display-board-panel、speaker-text-panel、display-board、speaker-text-area 等样式，控制尺寸与阴影。
- JS新增：voiceEnabled 全局变量与 toggleVoice() 函数，在 playAgentVoice() 前做开关判断。
- JS增强：addMessageToChat() 同步到 #speakerTextArea；showPresentation() 同步到 #displayBoardContent。

后续优化建议（可选）
- 将中部叠加层展示改为右上白板为主、中部用于座位态势与会议状态，仅在需要时叠加。
- 发言人文字展示区支持多条保留与自动轮播，提供“固定/滚动”模式切换。
- 资料白板支持 PPT/PPTX 的在线预览（可引入第三方服务或转换为PDF再展示）。
- 语音开关扩展为“系统TTS开关 + 麦克风采集开关”，并联动服务端音频通道。

验收要点
- 布局符合示意图意图，右上白板与右下文字区可用。
- 点击“语音开关”能明显关闭/开启系统朗读。
- 上传演示文稿并点击“展示”后，右上白板显示对应内容。
