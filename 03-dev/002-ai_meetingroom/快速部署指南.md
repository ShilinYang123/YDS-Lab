# AI开发团队可视化会议室 - 快速部署指南

## 🚀 一键启动

### 方法1: 使用Node.js直接启动
```bash
# 进入项目目录
cd ai_meeting_room

# 安装依赖
npm install

# 启动服务
node index.js
```

### 方法2: 使用启动脚本
```bash
# 进入项目目录
cd ai_meeting_room

# 赋予执行权限
chmod +x start.sh

# 启动服务
./start.sh
```

## 🌐 访问系统
打开浏览器访问: http://localhost:3000

## ✨ 主要功能

### 🎯 核心特性
- **可视化会议室**: 长桌式布局，直观的Agent位置显示
- **6个AI Agent**: 不同角色的开发团队成员
- **语音识别**: 每个Agent有独特的语音特征
- **实时聊天**: 支持多人实时交流
- **协作白板**: 实时共享会议记录
- **演示文稿**: 支持PPT/方案展示
- **移动端适配**: 支持手机参会

### 🧑‍💼 Agent角色
1. **前端开发工程师** (👨‍💻): React、TypeScript专家
2. **后端开发工程师** (👩‍💻): FastAPI、Python专家
3. **系统架构师** (🏗️): 微服务、云原生专家
4. **DevOps工程师** (🔧): CI/CD、容器化专家
5. **项目经理** (👨‍💼): 项目管理、团队协调
6. **测试工程师** (🧪): 自动化测试、质量保证

## 🎮 使用指南

### 📋 会议控制
- **开始会议**: 点击绿色"开始会议"按钮
- **结束会议**: 点击红色"结束会议"按钮
- **录音功能**: 点击橙色"录音"按钮

### 💬 聊天操作
- **发送消息**: 在输入框输入文字，点击"发送"按钮或按Ctrl+Enter
- **切换发言人**: 点击Agent头像或列表项

### 📝 白板使用
- **编辑内容**: 点击白板区域直接输入文字
- **实时同步**: 所有参会者实时看到更新

### 🎤 语音系统
每个Agent都有独特的语音特征：
- **male-clear**: 清晰的男性声音（前端开发）
- **female-professional**: 专业的女性声音（后端开发）
- **male-deep**: 深沉的男性声音（架构师）
- **male-technical**: 技术型男性声音（DevOps）
- **female-managerial**: 管理型女性声音（项目经理）
- **male-detailed**: 细致的男性声音（测试工程师）

## 🔧 技术配置

### 环境要求
- **Node.js**: 14.x 或更高版本
- **npm**: 6.x 或更高版本
- **浏览器**: Chrome 80+, Firefox 75+, Safari 14+, Edge 80+

### 端口配置
默认使用3000端口，可以通过环境变量修改：
```bash
PORT=8080 node index.js
```

### 生产环境部署
```bash
# 安装PM2进程管理
npm install -g pm2

# 启动应用
pm2 start index.js --name "ai-meeting-room"

# 设置开机自启
pm2 startup
pm2 save
```

## 🐛 常见问题

### Q1: 服务启动失败
**A**: 检查Node.js版本，确保安装了所有依赖
```bash
node --version
npm install
```

### Q2: 浏览器无法连接
**A**: 检查防火墙设置，确保3000端口开放
```bash
# Ubuntu系统
sudo ufw allow 3000
```

### Q3: 语音播放失败
**A**: 检查浏览器权限设置，确保允许音频播放
- Chrome: 设置 → 隐私和安全 → 网站设置 → 声音
- Firefox: 设置 → 隐私与安全 → 权限 → 自动播放声音

### Q4: 白板无法编辑
**A**: 检查是否有其他用户正在编辑，或刷新页面重试
