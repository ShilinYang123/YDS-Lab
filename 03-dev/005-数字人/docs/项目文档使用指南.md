## 🚀 快速开始

### 步骤1: 环境准备
确保您的系统已安装：
- Node.js 18+
- Docker 和 Docker Compose
- Git

### 步骤2: 一键初始化
根据您的操作系统选择对应的初始化脚本：

#### Windows 系统
```bash
# 在项目根目录执行
scripts\dev\setup.bat
```

#### macOS/Linux 系统
```bash
# 在项目根目录执行
chmod +x scripts/dev/setup.sh
./scripts/dev/setup.sh
```

### 步骤3: 启动开发环境
```bash
# 启动基础服务（PostgreSQL、Redis、MinIO）
docker-compose up -d postgres redis minio

# 启动前端开发服务器
cd src && npm run dev

# 启动后端API服务器
cd api && npm run dev
```

### 步骤4: 访问应用
- 前端应用: http://localhost:3000
- 后端API: http://localhost:3001
- MinIO控制台: http://localhost:9001

## 📋 技术栈总览

### 前端技术栈
- **框架**: React 18 + TypeScript
- **构建工具**: Vite
- **状态管理**: Zustand
- **UI组件**: Ant Design + Tailwind CSS
- **路由**: React Router v6
- **HTTP客户端**: Axios

### 后端技术栈
- **运行时**: Node.js + TypeScript
- **Web框架**: Express.js
- **数据库**: PostgreSQL (Supabase)
- **缓存**: Redis
- **文件存储**: MinIO
- **认证**: JWT

### 基础设施
- **容器化**: Docker + Docker Compose
- **反向代理**: Nginx
- **监控**: Prometheus + Grafana
- **日志**: Winston

## 📊 性能优化建议

### 前端优化
1. **代码分割**: 使用动态导入实现懒加载
2. **图片优化**: 压缩图片，使用WebP格式
3. **缓存策略**: 合理设置浏览器缓存
4. **CDN加速**: 使用CDN分发静态资源

### 后端优化
1. **数据库优化**: 建立合适的索引，优化查询
2. **缓存策略**: 使用Redis缓存热点数据
3. **连接池**: 合理配置数据库连接池
4. **异步处理**: 使用消息队列处理耗时任务

### 基础设施优化
1. **容器优化**: 使用多阶段构建减小镜像体积
2. **网络优化**: 使用HTTP/2，开启压缩
3. **监控告警**: 建立完善的监控体系
4. **日志管理**: 合理配置日志级别和轮转

## 🚨 常见问题解决

### Docker相关问题
```bash
# 容器无法启动
docker-compose logs <service-name>

# 端口冲突
netstat -tlnp | grep <port>

# 清理Docker资源
docker system prune -f
```

### 前端开发问题
```bash
# 依赖安装失败
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 构建失败
npm run build --verbose
```

### 后端开发问题
```bash
# 数据库连接失败
docker-compose exec postgres pg_isready -U postgres

# API测试失败
curl -X GET http://localhost:3001/health
```
